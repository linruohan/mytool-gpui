### 1. **完善错误处理体系** ⭐⭐⭐⭐⭐

#### 当前问题

- 部分代码使用 `unwrap()` 可能导致 panic
- 错误信息不够用户友好
- 缺少统一的错误处理策略

#### 优化方案

**Step 1: 定义统一的错误类型**

```rust
// crates/mytool/src/core/error.rs
use thiserror::Error;

#[derive(Error, Debug)]
pub enum AppError {
    #[error("数据库操作失败: {0}")]
    Database(#[from] todos::TodoError),

    #[error("序列化失败: {0}")]
    Serialization(#[from] serde_json::Error),

    #[error("配置错误: {0}")]
    Config(String),

    #[error("操作被取消")]
    Cancelled,

    #[error("未知错误: {0}")]
    Unknown(String),
}

impl AppError {
    /// 转换为用户友好的消息
    pub fn user_message(&self) -> String {
        match self {
            Self::Database(e) => format!("数据保存失败: {}", e.user_message()),
            Self::Serialization(_) => "数据格式错误，请重试".to_string(),
            Self::Config(msg) => format!("配置错误: {}", msg),
            Self::Cancelled => "操作已取消".to_string(),
            Self::Unknown(msg) => format!("发生错误: {}", msg),
        }
    }

    /// 判断是否可重试
    pub fn is_retryable(&self) -> bool {
        matches!(self, Self::Database(e) if e.is_retryable())
    }
}
```

**Step 2: 替换 unwrap() 为 Result**

```rust
// ❌ 之前
pub fn set_due(&mut self, due: DueDate) -> &mut Self {
    self.model.due = Some(serde_json::value::to_value(due).unwrap());
    self
}

// ✅ 之后
pub fn set_due(&mut self, due: DueDate) -> Result<&mut Self, AppError> {
    self.model.due = Some(serde_json::value::to_value(due)?);
    Ok(self)
}
```

**Step 3: 统一的错误通知机制**

```rust
// crates/mytool/src/core/state/error_notifier.rs (已存在，需增强)
pub struct ErrorNotifier {
    current_error: Option<AppError>,
    error_history: VecDeque<(Instant, AppError)>,
    max_history: usize,
}

impl ErrorNotifier {
    pub fn notify(&mut self, error: AppError) {
        // 记录错误历史
        self.error_history.push_back((Instant::now(), error.clone()));
        if self.error_history.len() > self.max_history {
            self.error_history.pop_front();
        }

        // 设置当前错误
        self.current_error = Some(error);

        // 触发 UI 更新
        // (GPUI 会自动通知订阅者)
    }

    pub fn get_user_message(&self) -> Option<String> {
        self.current_error.as_ref().map(|e| e.user_message())
    }
}
```

### 3. **增强操作队列机制** ⭐⭐⭐

#### 当前状态

- ✅ 已实现 `OperationQueue` (operation_queue.rs)
- ✅ 已实现 `OperationMerger` (operation_merger.rs)
- ⚠️ 未在实际业务中集成使用

#### 优化方案

**集成到乐观更新流程**:

```rust
// crates/mytool/src/core/actions/optimistic.rs

use super::operation_merger::OperationMerger;
use super::operation_queue::{OperationQueue, OpCommand};

// 全局操作合并器
static OPERATION_MERGER: Lazy<Mutex<OperationMerger>> = Lazy::new(|| {
    Mutex::new(OperationMerger::default())
});

// 全局操作队列
static OPERATION_QUEUE: Lazy<OperationQueue> = Lazy::new(|| {
    OperationQueue::new(ConflictStrategy::LastWriteWins)
});

pub fn update_item_optimistic(item: Arc<ItemModel>, cx: &mut App) {
    // 1. 立即更新 UI (乐观更新)
    cx.update_global::<TodoStore, _>(|store, _| {
        store.update_item(item.clone());
    });

    // 2. 队列化操作 (防抖和合并)
    let mut merger = OPERATION_MERGER.lock().unwrap();
    if merger.queue_update(item.clone()) {
        // 3. 启动防抖计时器
        let db = get_db_connection(cx);
        cx.spawn(async move |_cx| {
            // 等待防抖延迟
            tokio::time::sleep(Duration::from_millis(300)).await;

            // 4. 排空合并后的操作
            let operations = {
                let mut merger = OPERATION_MERGER.lock().unwrap();
                merger.drain_operations()
            };

            // 5. 提交到操作队列执行
            for op in operations {
                match op {
                    PendingOperation::Update(item) => {
                        let cmd = OpCommand::Update {
                            id: item.id.clone(),
                            item: item.clone(),
                            expected_version: item.version.unwrap_or(0) as usize,
                        };
                        let _ = OPERATION_QUEUE.submit(cmd, OperationMeta::default()).await;
                    }
                    _ => {}
                }
            }
        }).detach();
    }
}
```

### 4. **优化状态管理** ⭐⭐⭐

#### 当前问题

- 多个全局状态 (DBState, TodoStore, EventBus, QueryCache, etc.)
- 状态更新可能不同步

#### 优化方案

**引入状态版本控制**:

```rust
// crates/mytool/src/core/state/store.rs

pub struct TodoStore {
    // 现有字段...
    items: Vec<Arc<ItemModel>>,
    projects: Vec<Arc<ProjectModel>>,

    // 新增: 版本控制
    version: AtomicUsize,
    item_versions: HashMap<String, usize>,

    // 新增: 变更追踪
    dirty_items: HashSet<String>,
    last_sync: Instant,
}

impl TodoStore {
    pub fn update_item(&mut self, item: Arc<ItemModel>) {
        // 更新版本号
        let new_version = self.version.fetch_add(1, Ordering::SeqCst) + 1;
        self.item_versions.insert(item.id.clone(), new_version);

        // 标记为脏数据
        self.dirty_items.insert(item.id.clone());

        // 更新数据
        if let Some(pos) = self.items.iter().position(|i| i.id == item.id) {
            self.items[pos] = item;
        }
    }

    pub fn get_dirty_items(&self) -> Vec<String> {
        self.dirty_items.iter().cloned().collect()
    }

    pub fn mark_synced(&mut self, item_ids: &[String]) {
        for id in item_ids {
            self.dirty_items.remove(id);
        }
        self.last_sync = Instant::now();
    }
}
```

impl TodoStore {
    pub fn load_page(&mut self, view_id: &str, page: usize) -> Vec<Arc<ItemModel>> {
        let start = page * self.page_size;
        let end = start + self.page_size;

        self.items
            .iter()
            .skip(start)
            .take(self.page_size)
            .cloned()
            .collect()
